================================================================================
NAMING CONVENTIONS
================================================================================

ENTITIES:
  - Filename: lowercase, underscores for multi-word (matches table name)
  - Examples: users.md, order_items.md, user_preferences.md

ETL STEPS:
  - Pattern: step-{NN}-{descriptive-name}.md
  - NN: Two-digit number (01, 02, ... 10, 11)
  - Parallel steps: step-03a-name.md, step-03b-name.md
  - Examples: step-01-extract-legacy-data.md, step-05-transform-dates.md

API ENDPOINTS:
  - Pattern: {method}-{path-slug}.md
  - Method: lowercase (get, post, put, delete, patch)
  - Path: slashes become dashes, params simplified
  - Examples:
    - GET /api/users           → get-api-users.md
    - GET /api/users/:id       → get-api-users-id.md
    - POST /api/orders         → post-api-orders.md
    - DELETE /api/orders/:id   → delete-api-orders-id.md
    - GET /api/v2/products     → get-api-v2-products.md

OVERVIEW FILES:
  - Always prefixed with underscore: _overview.md, _pipeline-overview.md
  - Purpose: Index/summary files, not individual documentation
